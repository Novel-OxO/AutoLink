Pull Request를 한국어로 생성한다. `gh` CLI를 사용한다.

## 절차

1. 현재 브랜치와 커밋 이력을 확인한다 (`git log main..HEAD`)
2. `git diff main...HEAD`로 전체 변경 사항을 분석한다
3. 원격에 푸시되지 않은 커밋이 있으면 먼저 푸시한다
4. PR 제목과 본문을 한국어로 작성한다
5. `gh pr create`로 PR을 생성한다
6. 생성된 PR URL을 출력한다

## PR 제목 형식

커밋 컨벤션과 동일:

```
<type>(<scope>): <한국어 요약>
```

## PR 본문 형식

```markdown
## 관련 이슈

- closes #<이슈번호>

## 변경 사항

- [변경 내용 요약]

## 테스트

- [ ] 단위 테스트 통과
- [ ] 빌드 성공 확인
```

## 규칙

- 제목과 본문 모두 **한국어**로 작성
- base 브랜치는 `main`
- `closes #이슈번호`로 관련 이슈를 반드시 연결
- `--assignee @me` 기본 포함
- 브랜치명에서 이슈 번호를 자동 추출하여 연결한다

GitHub 이슈를 한국어로 생성한다. `gh` CLI를 사용한다.

## 절차

1. 사용자에게 이슈 유형을 질문한다 (기능/버그/문서/기타)
2. 이슈 제목과 상세 설명을 한국어로 작성한다
3. 적절한 라벨을 지정한다 (`enhancement`, `bug`, `docs`, `chore` 등)
4. `gh issue create`로 이슈를 생성한다
5. 생성된 이슈 번호와 URL을 출력한다

## 이슈 제목 형식

- 기능: `기능: [설명]`
- 버그: `버그: [설명]`
- 문서: `문서: [설명]`
- 기타: `[설명]`

## 규칙

- 제목과 본문 모두 **한국어**로 작성
- `--assignee @me` 기본 포함
- 생성 후 이슈 번호 기반 브랜치명을 제안한다 (예: `feat/#123-기능-설명`)

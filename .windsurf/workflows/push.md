현재 브랜치를 원격 저장소에 푸시한다. `git`을 사용한다.

## 절차

1. 현재 브랜치명을 확인한다
2. `pnpm lint`와 `pnpm build`로 오류가 없는지 확인한다
3. 원격 트래킹 브랜치 존재 여부를 확인한다
4. 최초 푸시면 `-u` 플래그를 포함하여 푸시한다
5. 푸시 결과를 출력한다

## 규칙

- `main` 브랜치에 직접 푸시하지 않는다 (main이면 경고 후 중단)
- `--force` 절대 금지. 필요 시 `--force-with-lease`만 사용하며 사용자 확인을 받는다
- 빌드/린트 실패 시 푸시하지 않고 문제를 알려준다

# AutoLink — 기능 명세서

> 사용자 행동(입력)과 시스템 반응(출력) 관점의 기능 명세

---

## 1. 현재 기능

### 1-1. 링크 저장

#### URL 직접 입력

| 항목 | 내용                                                                              |
| ---- | --------------------------------------------------------------------------------- |
| 입력 | 사용자가 URL을 입력하고 저장 버튼을 누른다                                        |
| 출력 | 해당 URL의 OG 메타데이터(제목, 설명, 썸네일)가 자동 추출되어 링크 카드로 저장된다 |
| 예외 | OG 메타데이터가 없는 경우 URL만 표시된다                                          |

#### iOS Share Extension

| 항목 | 내용                                                            |
| ---- | --------------------------------------------------------------- |
| 입력 | 사용자가 모바일 브라우저/앱에서 공유 버튼 → AutoLink를 선택한다 |
| 출력 | 해당 URL이 AutoLink에 저장되고, OG 메타데이터가 자동 추출된다   |

#### 크롬 익스텐션

| 항목 | 내용                                                               |
| ---- | ------------------------------------------------------------------ |
| 입력 | 사용자가 크롬에서 익스텐션 아이콘을 클릭한다                       |
| 출력 | 현재 탭의 URL이 AutoLink에 저장되고, OG 메타데이터가 자동 추출된다 |

### 1-2. 폴더 관리

#### 폴더 생성

| 항목 | 내용                                             |
| ---- | ------------------------------------------------ |
| 입력 | 사용자가 폴더 이름을 입력하고 생성 버튼을 누른다 |
| 출력 | 새 폴더가 생성되고 사이드바/폴더 목록에 표시된다 |

#### 링크를 폴더로 분류

| 항목 | 내용                                            |
| ---- | ----------------------------------------------- |
| 입력 | 사용자가 링크 카드를 드래그하여 폴더에 드롭한다 |
| 출력 | 해당 링크가 선택한 폴더로 이동된다              |

#### DOCK (폴더 고정)

| 항목 | 내용                                                        |
| ---- | ----------------------------------------------------------- |
| 입력 | 사용자가 폴더를 DOCK 영역에 고정한다                        |
| 출력 | 해당 폴더가 상단 DOCK 영역에 고정되어 빠르게 접근할 수 있다 |

### 1-3. 휴지통

#### 링크 삭제

| 항목 | 내용                                                          |
| ---- | ------------------------------------------------------------- |
| 입력 | 사용자가 링크를 삭제한다                                      |
| 출력 | 링크가 휴지통으로 이동한다. 30일간 보관 후 자동 영구 삭제된다 |

#### 링크 복원

| 항목 | 내용                                                   |
| ---- | ------------------------------------------------------ |
| 입력 | 사용자가 휴지통에서 링크를 선택하고 복원 버튼을 누른다 |
| 출력 | 해당 링크가 원래 위치로 복원된다                       |

### 1-4. AI 기능

#### 링크 자동 요약

| 항목 | 내용                                                          |
| ---- | ------------------------------------------------------------- |
| 입력 | 사용자가 링크를 저장한다                                      |
| 출력 | AI가 해당 링크의 콘텐츠를 분석하여 2~3줄 요약을 자동 생성한다 |

#### 자동 태깅

| 항목 | 내용                                                                                        |
| ---- | ------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 링크를 저장한다                                                                    |
| 출력 | AI가 콘텐츠를 분석하여 관련 키워드 태그를 자동으로 부여한다 (예: `NestJS`, `Guard`, `인증`) |

### 1-5. 공유

#### 폴더 공유

| 항목 | 내용                                                                      |
| ---- | ------------------------------------------------------------------------- |
| 입력 | 사용자가 폴더의 공유 버튼을 누른다                                        |
| 출력 | 공유 링크가 생성되고, 해당 링크를 통해 폴더 내 모든 링크를 열람할 수 있다 |

## 2. Phase 1 — 개인 지식 베이스 (RAG)

### 2-1. 콘텐츠 본문 크롤링

#### 자동 본문 수집

| 항목 | 내용                                                                                                       |
| ---- | ---------------------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 링크를 저장한다                                                                                   |
| 출력 | OG 메타데이터뿐 아니라 본문 전체가 백그라운드에서 자동 수집된다. 링크 카드에 "크롤링 완료" 상태가 표시된다 |
| 예외 | 크롤링 실패 시 "크롤링 실패" 상태가 표시되고, 사용자가 재시도할 수 있다                                    |

#### 다중 소스 대응

| 항목 | 내용                                                                                                                   |
| ---- | ---------------------------------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 다양한 출처의 URL을 저장한다 (블로그, 공식문서, YouTube, Stack Overflow 등)                                   |
| 출력 | 소스 유형에 맞게 본문이 추출된다. YouTube는 자막이 텍스트로 변환되고, Stack Overflow는 질문+답변이 구조적으로 수집된다 |

#### 중복 감지

| 항목 | 내용                                                                                |
| ---- | ----------------------------------------------------------------------------------- |
| 입력 | 사용자가 이미 저장된 URL을 다시 저장하려 한다                                       |
| 출력 | "이미 저장된 링크입니다"라는 안내와 함께 기존 링크로 이동할 수 있는 옵션이 제공된다 |

#### 콘텐츠 변경 감지

| 항목 | 내용                                                                                         |
| ---- | -------------------------------------------------------------------------------------------- |
| 입력 | 저장된 링크의 원본 페이지가 업데이트된다 (시스템이 주기적으로 확인)                          |
| 출력 | "원본 업데이트됨" 배지가 표시되고, 사용자가 클릭하면 변경 내역을 확인하고 재크롤링할 수 있다 |

### 2-2. 시맨틱 검색

#### 자연어 검색

| 항목 | 내용                                                                                                                |
| ---- | ------------------------------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 검색창에 자연어를 입력한다 (예: "NestJS에서 커스텀 가드 만드는 방법")                                      |
| 출력 | 의미적으로 관련된 링크들이 유사도 순으로 정렬되어 표시된다. 각 결과에는 제목, 요약, 관련 구간 하이라이트가 포함된다 |

#### 하이브리드 검색

| 항목 | 내용                                                                                                      |
| ---- | --------------------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 검색어를 입력한다                                                                                |
| 출력 | 시맨틱 검색(의미 기반)과 키워드 검색(정확한 단어 매칭)이 결합되어 가장 관련도 높은 결과가 상위에 노출된다 |

### 2-3. RAG 질의응답

#### 자연어 질문

| 항목 | 내용                                                                                                                        |
| ---- | --------------------------------------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 채팅 UI에서 질문을 입력한다 (예: "NestJS Guard 커스텀 방법 뭐였지?")                                               |
| 출력 | 저장된 자료를 기반으로 AI가 답변을 실시간 스트리밍으로 생성한다. 답변 하단에 참고한 링크 목록과 해당 구간이 출처로 표시된다 |

#### 대화 이어가기

| 항목 | 내용                                                                              |
| ---- | --------------------------------------------------------------------------------- |
| 입력 | 사용자가 이전 답변에 이어서 추가 질문을 한다 (예: "그러면 Role 기반으로 하려면?") |
| 출력 | 이전 대화 맥락을 유지한 상태에서 추가 답변이 생성된다                             |

#### 메모 통합 검색

| 항목 | 내용                                                                    |
| ---- | ----------------------------------------------------------------------- |
| 입력 | 사용자가 질문한다                                                       |
| 출력 | 링크 본문뿐 아니라 사용자가 남긴 메모 내용까지 포함하여 답변이 생성된다 |

## 3. Phase 2 — 학습 가속기

### 3-1. 일일 추천

#### 오늘의 추천 링크

| 항목 | 내용                                                                                                                       |
| ---- | -------------------------------------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 앱을 열거나 홈 화면에 접근한다                                                                                    |
| 출력 | "오늘의 추천" 카드가 표시된다. 최근 저장한 주제의 흐름, 현재 수준에 맞는 난이도, 안 읽은 링크 등을 고려하여 1개가 선정된다 |

#### 미읽은 링크 리마인드

| 항목 | 내용                                                |
| ---- | --------------------------------------------------- |
| 입력 | 사용자가 링크를 저장한 후 3일 이상 열어보지 않았다  |
| 출력 | "아직 안 읽은 링크" 알림이 표시되어 읽기를 유도한다 |

#### 균형 추천

| 항목 | 내용                                                                            |
| ---- | ------------------------------------------------------------------------------- |
| 입력 | 사용자가 한 주제(예: 프론트엔드)만 집중적으로 저장/열람한다                     |
| 출력 | 다른 영역(예: 인프라, DB)의 저장된 자료 중 추천이 포함되어 학습 균형을 유도한다 |

### 3-2. 학습 시각화 대시보드

#### 주제별 히트맵

| 항목 | 내용                                                                                         |
| ---- | -------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 대시보드 화면에 접근한다                                                            |
| 출력 | 어떤 분야의 자료를 얼마나 읽었는지 히트맵 형태로 시각화된다 (GitHub 잔디처럼 색 농도로 표현) |

#### 지식 레이더 차트

| 항목 | 내용                                                                            |
| ---- | ------------------------------------------------------------------------------- |
| 입력 | 사용자가 대시보드 화면에 접근한다                                               |
| 출력 | 프론트엔드, 백엔드, 인프라, DB 등 영역별 학습 커버리지가 레이더 차트로 표시된다 |

#### 기술 타임라인

| 항목 | 내용                                                                   |
| ---- | ---------------------------------------------------------------------- |
| 입력 | 사용자가 대시보드 화면에 접근한다                                      |
| 출력 | 월별 학습 주제 변화가 타임라인으로 표시되어 성장 궤적을 확인할 수 있다 |

#### 읽기 패턴 분석

| 항목 | 내용                                                                              |
| ---- | --------------------------------------------------------------------------------- |
| 입력 | 사용자가 대시보드 화면에 접근한다                                                 |
| 출력 | 선호하는 콘텐츠 유형(블로그, 공식문서, 영상 등)과 주로 학습하는 시간대가 표시된다 |

### 3-3. 인사이트 자동 축적

#### 반복 질문 감지

| 항목 | 내용                                                                                                                              |
| ---- | --------------------------------------------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 같은 주제에 대해 3번 이상 질문한다                                                                                       |
| 출력 | 해당 주제가 "약점"으로 분류되고, 관련 심화 자료가 추천된다. "이 주제를 자주 찾으시네요. 심화 자료를 확인해보세요" 안내가 표시된다 |

#### 저장-활용 전환율

| 항목 | 내용                                                                   |
| ---- | ---------------------------------------------------------------------- |
| 입력 | 저장된 링크 중 한 번도 열어보거나 질의응답에 활용되지 않은 링크가 있다 |
| 출력 | "저장만 하고 활용하지 않은 링크 N개" 알림이 주기적으로 표시된다        |

#### 질문 로그 분석

| 항목 | 내용                                          |
| ---- | --------------------------------------------- |
| 입력 | 사용자가 대시보드에 접근한다                  |
| 출력 | 자주 질문한 주제 TOP 5와 질문 빈도가 표시된다 |

#### 주간 회고 자동 생성

| 항목 | 내용                                                              |
| ---- | ----------------------------------------------------------------- |
| 입력 | 매주 일정 시점이 되면 자동으로 생성된다 (또는 사용자가 수동 요청) |
| 출력 | 이번 주 저장한 링크, 질문한 내용, 학습 요약이 정리되어 표시된다   |

## 4. Phase 3 — 세컨드 브레인

### 4-1. 지식 연결

#### 자동 연결 발견

| 항목 | 내용                                                                                                                             |
| ---- | -------------------------------------------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 새 링크를 저장한다                                                                                                      |
| 출력 | 기존에 저장된 링크 중 의미적으로 관련된 자료가 "관련 링크" 섹션에 자동 표시된다 (예: 3개월 전 저장한 글과 오늘 저장한 글의 연결) |

#### 크로스 폴더 추천

| 항목 | 내용                                                                                                         |
| ---- | ------------------------------------------------------------------------------------------------------------ |
| 입력 | 사용자가 특정 링크를 열람한다                                                                                |
| 출력 | 다른 폴더에 있지만 의미적으로 관련된 자료가 추천된다. "다른 폴더에서 관련 자료를 발견했어요" 형태로 표시된다 |

#### 지식 그래프

| 항목 | 내용                                                                                                          |
| ---- | ------------------------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 지식 그래프 화면에 접근한다                                                                          |
| 출력 | 저장된 링크와 메모 간의 관계가 노드-엣지 그래프로 시각화된다. 노드를 클릭하면 해당 링크/메모로 이동할 수 있다 |

### 4-2. 외부 연동

#### GitHub 연동

| 항목 | 내용                                                                                                                                          |
| ---- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 GitHub 계정을 연동한다                                                                                                               |
| 출력 | 최근 커밋/PR에서 사용된 기술 키워드가 자동 추출되고, 관련 저장 자료가 추천된다 (예: 최근 Redis 관련 코드를 작성했다면 저장된 Redis 자료 추천) |

#### 코드 리뷰 피드백 저장

| 항목 | 내용                                                                                  |
| ---- | ------------------------------------------------------------------------------------- |
| 입력 | 사용자가 받은 코드 리뷰 피드백을 저장한다                                             |
| 출력 | 이후 비슷한 패턴의 코드를 다룰 때 "이전에 이런 리뷰를 받은 적 있어요" 알림이 표시된다 |

#### RSS/뉴스레터 연동

| 항목 | 내용                                                                                   |
| ---- | -------------------------------------------------------------------------------------- |
| 입력 | 사용자가 RSS 피드 URL 또는 뉴스레터 주소를 등록한다                                    |
| 출력 | 새 글이 발행되면 자동 수집되고, 사용자의 관심사와 일치하는 글만 필터링되어 알림이 온다 |

#### 면접 준비 모드

| 항목 | 내용                                                                                                                   |
| ---- | ---------------------------------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 면접 준비 모드를 활성화한다                                                                                   |
| 출력 | 저장된 자료를 기반으로 맞춤 면접 질문이 자동 생성된다. 사용자가 답변하면 저장한 자료와 비교하여 부족한 부분을 짚어준다 |

## 5. Phase 4 — 개발자 지식 플랫폼

### 5-1. 공개/비공개 시스템

#### 링크 공개 설정

| 항목 | 내용                                                                            |
| ---- | ------------------------------------------------------------------------------- |
| 입력 | 사용자가 링크의 공개 범위를 설정한다 (비공개 / 공개)                            |
| 출력 | 공개로 설정된 링크는 다른 사용자가 검색하고 열람할 수 있다. 비공개가 기본값이다 |

#### 폴더 단위 공개

| 항목 | 내용                                                                              |
| ---- | --------------------------------------------------------------------------------- |
| 입력 | 사용자가 폴더 전체를 공개로 설정한다                                              |
| 출력 | 해당 폴더와 포함된 링크가 모두 공개되어 다른 사용자가 큐레이션으로 열람할 수 있다 |

#### 프로필 공개

| 항목 | 내용                                                              |
| ---- | ----------------------------------------------------------------- |
| 입력 | 사용자가 프로필 공개를 활성화한다                                 |
| 출력 | 사용자의 공개 큐레이션 전체를 볼 수 있는 프로필 페이지가 생성된다 |

### 5-2. 워크스페이스

#### 워크스페이스 생성

| 항목 | 내용                                                                  |
| ---- | --------------------------------------------------------------------- |
| 입력 | 사용자가 워크스페이스 이름과 용도를 입력하고 생성한다                 |
| 출력 | 독립된 지식 베이스(링크, 폴더, RAG)를 가진 새 워크스페이스가 생성된다 |

#### 워크스페이스 전환

| 항목 | 내용                                                                                                    |
| ---- | ------------------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 워크스페이스 선택 UI에서 다른 워크스페이스를 클릭한다                                          |
| 출력 | 화면이 선택한 워크스페이스로 전환된다. 링크, 폴더, 검색, RAG가 모두 해당 워크스페이스의 데이터로 바뀐다 |

#### 멤버 초대

| 항목 | 내용                                                                                        |
| ---- | ------------------------------------------------------------------------------------------- |
| 입력 | 관리자가 초대할 사용자의 이메일을 입력하고 초대 버튼을 누른다                               |
| 출력 | 초대 이메일이 발송된다. 초대받은 사용자가 가입/로그인하면 해당 워크스페이스에 자동 참여된다 |

#### 워크스페이스 RAG

| 항목 | 내용                                                                   |
| ---- | ---------------------------------------------------------------------- |
| 입력 | 사용자가 워크스페이스에서 질문한다                                     |
| 출력 | 해당 워크스페이스에 저장된 모든 멤버의 자료를 기반으로 답변이 생성된다 |

#### 권한 관리

| 항목 | 내용                                                                         |
| ---- | ---------------------------------------------------------------------------- |
| 입력 | 관리자가 멤버의 권한을 설정한다                                              |
| 출력 | 관리자: 멤버 초대/제거, 설정 변경 가능. 멤버: 링크 저장, RAG 질의, 열람 가능 |

#### 크로스 워크스페이스 검색

| 항목 | 내용                                                                 |
| ---- | -------------------------------------------------------------------- |
| 입력 | 사용자가 검색 시 "전체 워크스페이스" 옵션을 선택한다                 |
| 출력 | 사용자가 속한 모든 워크스페이스의 자료에서 통합 검색 결과가 표시된다 |

### 5-3. 커뮤니티

#### 큐레이션 구독

| 항목 | 내용                                                           |
| ---- | -------------------------------------------------------------- |
| 입력 | 사용자가 다른 개발자의 공개 프로필/폴더에서 구독 버튼을 누른다 |
| 출력 | 구독한 개발자가 새 링크를 공개하면 알림을 받는다               |

#### 인기 링크 트렌드

| 항목 | 내용                                                         |
| ---- | ------------------------------------------------------------ |
| 입력 | 사용자가 트렌드 피드에 접근한다                              |
| 출력 | 가장 많이 저장된 링크, 요즘 뜨는 주제가 피드 형태로 표시된다 |

#### 지식 프로필

| 항목 | 내용                                                                                       |
| ---- | ------------------------------------------------------------------------------------------ |
| 입력 | 사용자가 공개 프로필을 활성화한 상태에서 활동을 지속한다                                   |
| 출력 | 저장/공유 패턴을 분석하여 "이 개발자는 이런 주제에 강합니다" 형태의 자동 프로필이 생성된다 |

#### 집단 지성 답변

| 항목 | 내용                                                                                           |
| ---- | ---------------------------------------------------------------------------------------------- |
| 입력 | 사용자가 질문했으나 개인 자료에 관련 내용이 부족하다                                           |
| 출력 | 커뮤니티의 공개 자료에서 관련 내용을 찾아 답변에 포함한다. 출처에 "커뮤니티 자료"임이 표시된다 |

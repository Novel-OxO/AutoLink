# AutoLink — Evolution Roadmap

> 북마크 앱에서 개발자 지식 플랫폼으로 | 2025.02 - 2026

---

## 1. 제품 비전

AutoLink는 단순한 북마크 앱이 아닙니다.
개발자가 읽고, 이해하고, 정리하고, 성장하는 전 과정을 하나의 플랫폼에서 완성하는 **"개발자의 세컨드 브레인"**입니다.

### 핵심 가치

- **저장이 아닌 이해**: 링크를 모으는 게 아니라, 모은 지식을 연결하고 활용하게 만든다
- **수동이 아닌 자동**: AI가 요약, 태깅, 연결, 추천을 대신 해준다
- **소비가 아닌 생산**: 읽기에서 끝나지 않고, 지식 연결과 활용까지 이어지는 학습 루프
- **개인이 아닌 커뮤니티**: 공개된 지식이 다른 개발자의 학습 자료가 되는 선순환

### 제품 진화 단계

| 단계    | 포지셔닝           | 핵심 가치                         |
| ------- | ------------------ | --------------------------------- |
| 현재    | 링크 저장/관리 앱  | 빠른 저장, OG 미리보기, 폴더 정리 |
| Phase 1 | 개인 지식 베이스   | RAG 질의응답, 시맨틱 검색         |
| Phase 2 | 학습 가속기        | 일일 추천, 학습 시각화, 약점 분석 |
| Phase 3 | 세컨드 브레인      | 지식 연결, 외부 연동, 회고 자동화 |
| Phase 4 | 개발자 지식 플랫폼 | 공개/비공개, 커뮤니티, 집단 지성  |

## 3. Phase 1 — 개인 지식 베이스 (RAG)

저장한 링크에 "질문"할 수 있게 만드는 단계입니다. 이 단계가 완성되면 사용자가 링크를 저장하는 이유가 근본적으로 달라집니다.

### 3-1. 콘텐츠 크롤링 파이프라인

링크를 저장하면 본문 전체를 자동으로 크롤링하고 파싱합니다.

| 기능             | 상세                                           | 기술 스택            |
| ---------------- | ---------------------------------------------- | -------------------- |
| 본문 전체 크롤링 | OG 메타데이터를 넘어 글 전체를 수집            | Playwright + Cheerio |
| 다중 소스 대응   | 블로그, 공식문서, YouTube 자막, Stack Overflow | 소스별 파서 분기     |
| 동적 렌더링 대응 | SPA, CSR 페이지 크롤링                         | Playwright headless  |
| 중복 감지        | URL 해시 + 콘텐츠 해시로 중복 등록 방지        | SHA-256 해시         |
| 콘텐츠 변경 감지 | 저장한 문서의 업데이트 자동 감지 및 재크롤링   | Cron + diff 비교     |

### 3-2. 임베딩 파이프라인

크롤링한 콘텐츠를 벡터화하여 시맨틱 검색이 가능하게 만듭니다.

| 기능            | 상세                                             | 기술 스택                   |
| --------------- | ------------------------------------------------ | --------------------------- |
| 적응형 청킹     | 콘텐츠 타입별 최적 청킹 전략 자동 적용           | 커스텀 청커                 |
| 벡터 임베딩     | 텍스트를 벡터로 변환                             | OpenAI Embedding API        |
| 벡터 저장/검색  | 유사도 기반 시맨틱 검색                          | PostgreSQL + pgvector       |
| 하이브리드 검색 | 시맨틱 검색 + 키워드 검색 결합                   | pgvector + Full-text Search |
| 비동기 처리     | 링크 등록 시 크롤링/임베딩을 큐 기반 비동기 처리 | Redis + Bull Queue          |

### 3-3. RAG 질의응답

사용자가 자연어로 질문하면 저장된 자료를 기반으로 답변합니다.

| 기능           | 상세                                  | 기술 스택               |
| -------------- | ------------------------------------- | ----------------------- |
| 자연어 질의    | "NestJS Guard 커스텀 방법 뭐였지?"    | LLM API (Claude/OpenAI) |
| 출처 표시      | 답변에 참고한 링크와 해당 구간 표시   | 청크 메타데이터         |
| 스트리밍 응답  | 실시간으로 답변이 생성되는 채팅 UI    | SSE / WebSocket         |
| 대화 맥락 유지 | 이전 질문을 이어서 추가 질문 가능     | 대화 히스토리 관리      |
| 메모 통합 검색 | 내가 남긴 메모까지 포함해서 검색/답변 | 메모 임베딩 통합        |

## 4. Phase 2 — 학습 가속기

저장한 지식을 실제 학습으로 전환하는 기능입니다. 쓰면 쓸수록 나에 대한 데이터가 쌓이고, 그 데이터가 다시 나를 성장시킵니다.

### 4-1. 일일 추천

매일 "오늘의 1개"를 골라주되, 내 학습 맥락을 고려한 스마트 추천을 제공합니다.

| 기능             | 상세                                             |
| ---------------- | ------------------------------------------------ |
| 맥락 기반 추천   | 최근 저장한 주제의 흐름을 이어가는 관련 글 추천  |
| 균형 추천        | 한 주제만 편식할 때 다른 영역 탐색 유도          |
| 난이도 조절      | 현재 수준에 맞는 글 우선 추천 (입문 → 심화 순서) |
| 미읽은 링크 우선 | 저장 후 3일 이상 안 읽은 링크 리마인드           |

### 4-2. 학습 시각화 대시보드

내가 뭘 알고 뭘 모르는지를 데이터로 보여줍니다.

| 기능             | 상세                                              |
| ---------------- | ------------------------------------------------- |
| 주제별 히트맵    | 어떤 분야의 자료를 얼마나 읽었는지 시각화         |
| 지식 레이더 차트 | 프론트엔드, 백엔드, 인프라, DB 등 영역별 커버리지 |
| 기술 타임라인    | 월별 학습 주제 변화 추적 (성장 궤적)              |
| 읽기 패턴 분석   | 선호하는 콘텐츠 유형, 학습 시간대 분석            |

### 4-3. 인사이트 자동 축적

사용할수록 자동으로 쌓이는 나만의 학습 데이터입니다.

| 기능                | 상세                                                        |
| ------------------- | ----------------------------------------------------------- |
| 반복 질문 감지      | 같은 주제를 3번 이상 질문하면 약점으로 분류, 심화 자료 추천 |
| 저장-활용 전환율    | 저장만 하고 한 번도 활용 안 된 링크 알림                    |
| 질문 로그 분석      | 어떤 주제를 자주 질문하는지 패턴 추출                       |
| 주간 회고 자동 생성 | 이번 주 읽은 것, 질문한 것, 학습 요약을 자동 정리           |

## 5. Phase 3 — 세컨드 브레인

저장한 지식을 자동으로 연결하고, 외부 개발 흐름과 통합하여 학습의 완전한 사이클을 완성합니다.

### 5-1. 지식 연결

사용자가 모르는 연결을 시스템이 자동으로 발견합니다.

| 기능             | 상세                                                   |
| ---------------- | ------------------------------------------------------ |
| 자동 연결 발견   | 3개월 전 저장한 글과 오늘 저장한 글의 관련성 자동 감지 |
| 크로스 폴더 추천 | 다른 폴더에 있지만 의미적으로 관련된 자료 연결         |
| 지식 그래프      | 링크, 메모, 블로그 글 간의 관계를 그래프로 시각화      |

### 5-2. 외부 연동

개발자의 실제 작업 흐름과 연결됩니다.

| 기능                  | 상세                                                     |
| --------------------- | -------------------------------------------------------- |
| GitHub 연동           | 최근 작성한 코드의 기술 키워드 자동 추출, 관련 자료 추천 |
| 코드 리뷰 피드백 저장 | 받은 리뷰를 저장하면 같은 실수 반복 시 알림              |
| RSS/뉴스레터 연동     | 자동 수집 후 관심사 기반 필터링                          |
| 면접 준비 모드        | 내 자료 기반 맞춤 면접 질문 자동 생성                    |

## 6. Phase 4 — 개발자 지식 플랫폼

개인 도구에서 커뮤니티 플랫폼으로 확장합니다. 한 사람의 지식이 다른 사람의 학습 자료가 되는 선순환 구조를 만듭니다.

### 6-1. 공개/비공개 시스템

| 공개 범위      | 설명                                                 |
| -------------- | ---------------------------------------------------- |
| 비공개 (기본)  | 모든 콘텐츠는 비공개가 기본, 부담 없이 자유롭게 정리 |
| 링크 단위 공개 | 특정 링크 + 메모를 공개                              |
| 폴더 단위 공개 | 큐레이션된 폴더를 통째로 공유                        |
| 프로필 공개    | 내 큐레이션 전체를 외부에 공개                       |

### 6-2. 워크스페이스

하나의 계정에서 팀/회사/스터디별로 지식 베이스를 완전히 분리합니다. Slack의 워크스페이스처럼, 맥락에 따라 전환하며 사용합니다.

| 기능                     | 상세                                                                             |
| ------------------------ | -------------------------------------------------------------------------------- |
| 워크스페이스 생성        | 개인용, 회사용, 스터디용 등 목적별 지식 베이스 분리                              |
| 독립된 RAG               | 워크스페이스마다 별도의 임베딩/벡터 DB, 질문하면 해당 워크스페이스 자료만 검색   |
| 이메일 초대              | Notion 방식으로 이메일 초대 → 가입 필수 → 워크스페이스 참여                      |
| 권한 관리 (2단계)        | 관리자: 멤버 초대/제거, 워크스페이스 설정 변경 / 멤버: 링크 저장, RAG 질의, 열람 |
| 워크스페이스 전환        | 한 번의 클릭으로 개인 ↔ 회사 ↔ 스터디 전환                                       |
| 워크스페이스별 대시보드  | 각 워크스페이스의 학습 시각화가 독립적으로 제공                                  |
| 크로스 워크스페이스 검색 | 필요시 여러 워크스페이스를 동시에 검색하는 옵션                                  |

### 6-3. 커뮤니티 플라이휠

콘텐츠가 플랫폼 안에서 순환하며 가치가 증폭됩니다.

| 기능             | 상세                                                 |
| ---------------- | ---------------------------------------------------- |
| 큐레이션 구독    | 다른 개발자의 공개 폴더/큐레이션을 구독              |
| 구독 알림 시스템 | 구독한 유저가 새 링크를 공개하면 푸시/인앱 알림 수신 |
| 인기 링크 트렌드 | 가장 많이 저장된 링크, 요즘 뜨는 주제 피드           |
| 지식 프로필      | 어떤 주제에 강한 개발자인지 자동 프로필 생성         |
| 팀 지식 베이스   | 팀원들의 링크가 합쳐진 공유 RAG                      |
| 집단 지성 답변   | 내 자료에 없는 질문도 커뮤니티 자료에서 답변         |

## 7. 수익 모델 (Pricing)

기본 단위는 워크스페이스입니다. 모든 유저는 가입 시 개인 워크스페이스를 갖고, Pro 구독 시 AI 기능 해금 + 멤버 초대가 가능해집니다. 구독 등급에 따라 초대 가능 인원과 AI 사용량이 늘어나는 구조입니다.

### Free

개인 워크스페이스 1개로 시작합니다. 공개 링크를 쌓아 플랫폼 콘텐츠 풀을 만드는 역할입니다.

| 기능                     | 제한        |
| ------------------------ | ----------- |
| 개인 워크스페이스        | 1개         |
| 퍼블릭 링크 저장         | 무제한      |
| OG 크롤링 / 자동 태깅    | 제공        |
| 폴더 정리 / 드래그앤드롭 | 제공        |
| 공개 링크 검색 / 트렌드  | 제공        |
| 다른 유저 구독           | 제공        |
| 프라이빗 링크            | 월 30개     |
| AI 채팅 (RAG)            | 미제공      |
| 멤버 초대                | 미제공      |
| 학습 대시보드            | 기본만 제공 |

### Pro

AI 기능이 해금되고, 워크스페이스에 소수 인원을 초대할 수 있습니다. 개인 또는 소규모 스터디에 적합합니다.

| 기능                   | 상세                                   |
| ---------------------- | -------------------------------------- |
| 워크스페이스           | 최대 3개                               |
| 멤버 초대              | 워크스페이스당 최대 5명                |
| 프라이빗 링크          | 무제한                                 |
| AI 채팅 (RAG 질의응답) | 내 자료 기반 시맨틱 검색 + 대화형 답변 |
| 일일 추천 / 약점 분석  | 학습 패턴 기반 스마트 추천             |
| 학습 대시보드 전체     | 히트맵, 레이더 차트, 타임라인 등       |
| 주간 회고 자동 생성    | 학습 요약 자동 정리                    |
| 면접 준비 모드         | 내 자료 기반 맞춤 면접 질문 생성       |

### Team

대규모 팀과 회사를 위한 플랜입니다. 워크스페이스와 멤버 수 제한이 확장되고, 팀 전용 기능이 추가됩니다.

| 기능                     | 상세                                  |
| ------------------------ | ------------------------------------- |
| 워크스페이스             | 무제한                                |
| 멤버 초대                | 워크스페이스당 최대 50명              |
| Pro 전체 기능            | AI 채팅, 대시보드, 추천 등 모두 포함  |
| 팀 RAG                   | 팀원들의 링크가 합쳐진 공유 질의응답  |
| 팀 대시보드              | 팀 전체의 학습 현황 시각화            |

### 8-1. i18n (다국어)

DB 기반 번역 시스템으로 한국어와 영어를 지원합니다. UI 텍스트, 시스템 메시지, 알림 등 모든 사용자 노출 텍스트가 번역 대상입니다.

| 항목           | 내용                                                    |
| -------------- | ------------------------------------------------------- |
| 지원 언어      | 한국어 (ko), 영어 (en)                                  |
| 기본 언어      | 한국어 (ko)                                             |
| 언어 결정 순서 | 유저 설정 → 브라우저/디바이스 언어 → 기본 언어          |
| 번역 저장      | DB `translations` 테이블에 key-locale-value 구조로 저장 |
| 번역 대상      | UI 텍스트, 시스템 메시지, 알림 메시지, 에러 메시지      |
| 번역 미존재 시 | 기본 언어(ko) 텍스트로 폴백                             |

### 8-2. 타입 안전성 아키텍처

DB부터 UI까지 타입이 끊김 없이 관통하는 구조입니다. Prisma가 DB 스키마에서 타입을 생성하고, Zod 스키마가 백엔드와 프론트엔드를 연결합니다.

**타입 흐름**: `PostgreSQL → Prisma (타입 자동생성) → NestJS DTO (Zod 검증) → packages/shared (Zod 스키마) → Next.js + TanStack Query (응답 파싱) → UI 컴포넌트`

| 구간            | 타입 보장 방식              | 이점                                             |
| --------------- | --------------------------- | ------------------------------------------------ |
| DB → 백엔드     | Prisma Client 자동생성 타입 | 스키마 변경 시 컴파일 에러로 즉시 감지           |
| 백엔드 내부     | NestJS DTO + Zod validation | 런타임 검증 + 컴파일 타임 타입 체크 동시         |
| 백엔드 → 프론트 | packages/shared Zod 스키마  | API 응답 타입을 한 곳에서 관리, 양쪽 자동 동기화 |
| 프론트 내부     | TanStack Query + Zod parse  | 서버 응답을 Zod로 파싱하여 런타임 안전성 확보    |

## 9. 핵심 지표 (Key Metrics)

### 북극성 지표 (North Star Metric)

**주간 질의 수 (Weekly Queries)** — 사용자가 일주일에 몇 번 AutoLink에 질문하는가. 저장 횟수가 아니라 질문 횟수가 핵심입니다. 저장만 하는 서비스는 잊히지만, 질문하는 서비스는 매일 돌아오게 됩니다.
